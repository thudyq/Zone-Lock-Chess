# 封域棋 Zone Lock Chess

**在线体验**：可直接访问 [https://thudyq.github.io/Zone-Lock-Chess/](https://thudyq.github.io/Zone-Lock-Chess/) 开始游戏，无需任何安装。

**封域棋** 是一款基于原创规则的双人策略棋类游戏，支持本地双人对战、人机对战以及在线多人对战。游戏围绕“限制对手行动空间”的核心机制展开，规则简洁但富有深度。

## 📜 游戏规则

### 基本规则
- 游戏在 `N × N` 的棋盘上进行（支持 6×6、8×8、10×10、12×12）。
- 两名玩家分别执黑棋（先手）和白棋（后手），轮流在空格上落子。
- 棋子落下后不可移动，同一格只能落子一次。

### 落子约束（核心规则）
某个空格对当前玩家合法，当且仅当该格子的**上下左右四个方向**均不存在对方棋子。换句话说：
- 棋子可以紧挨己方棋子落子。
- 棋子**不能**紧挨对方棋子落子。
- 若一个空格同时与双方棋子相邻（即上下左右既有黑棋也有白棋），则对该格 **双方均不可落子**，该格成为永久废点。

### 胜负判定
游戏在任一玩家无合法落子位置时立即结束。此时：
- 若另一方仍有合法落子位置，则**该方获胜**。
- 若双方均无合法落子位置，则**平局**。

## 🎮 功能特性

### 核心游戏
- **动态棋盘**：支持 6×6、8×8、10×10、12×12 四种尺寸，棋盘大小可在游戏过程中随时切换（在线模式下需双方确认）。
- **回合制对战**：自动切换黑白双方，实时显示当前回合和可落子步数。
- **合法落点高亮**：当前玩家的所有合法落子位置会以绿色光点提示，悬停时显示半透明棋子预览。
- **局势评估**：基于双方可行动作数差值实时显示“黑棋优势”、“白棋优势”或“均势”，帮助玩家判断局面。
- **游戏结束弹窗**：胜负或平局时显示醒目弹窗，提供“再来一局”和“导出棋谱”快捷操作。

### 人机对战（AI）
AI 系统提供四种难度等级，满足不同水平玩家的需求：
- **简单**：完全随机落子，适合新手熟悉规则。
- **中等**：采用混合启发式评估函数（综合可行动作数、中心控制、集团密度），选择最优落点。
- **困难**：基于 Minimax 算法 + Alpha-Beta 剪枝，搜索深度为 2 层，模拟对手应对。
- **专家**：搜索深度提升至 3 层，策略性更强，适合挑战。

玩家可以自由选择执黑或执白，AI 会自动适应并延迟响应以模拟思考过程。

### 棋谱系统
- **历史记录**：每一步落子都会记录在右侧棋谱列表中，并自动滚动至最新一步。
- **导出棋谱**：以 JSON 格式导出完整棋局，包含棋盘尺寸和所有落子记录，文件名自动附带时间戳。
- **导入棋谱**：加载已保存的 JSON 文件，自动进入回放模式，便于复盘分析。
- **棋局回放**：提供完整的回放控制（⏮ 开始、⏴ 上一步、▶/⏸ 播放/暂停、⏵ 下一步、⏭ 结束），可调节播放速度（慢/正常/快），回放时当前步在棋谱列表中高亮显示，并允许随时从当前步恢复对弈。

### 在线对战
基于 Node.js 服务器实现实时双人对战，支持：
- **创建房间**：生成 6 位房间号（含字母和数字），房主可分享给对手。
- **加入房间**：输入房间号即可加入，双方进入“颜色选择”阶段，自由选择执黑或执白。
- **实时同步**：通过轮询机制（每 250ms）同步棋盘状态，确保双方所见一致。
- **协同操作**：棋盘大小修改、重新开始、悔棋等功能均需**双方确认**方可生效，防止一方擅自操作。
- **游戏结束后**，任何一方可发起“再来一局”，对方同意后重置棋盘。
- **状态提示**：清晰显示当前回合、轮到谁落子、网络状态、房间号等关键信息。
- **离开处理**：当一方离开时，另一方会收到提示，房间自动解散或转为等待状态。

#### 📡 联机指南

在线对战需要其中一位玩家（**房主**）启动本地服务器，另一位玩家（**访客**）通过网络连接。根据网络环境，推荐以下方案（按推荐度排序）：

| 你的网络环境 | 推荐方案 |
| :--- | :--- |
| 校内校园网 / 教育网（支持 IPv6） | **方案一：直连 IPv6**（无需任何工具） |
| 校外有 VPN 账号（如校园 VPN） | **方案二：VPN 内网直连**（稳定可靠） |
| 双方都在 NAT 后，无公网 IP | **方案三：Tailscale 虚拟组网**（免费，5 分钟） |
| 不想安装任何软件，愿使用第三方服务 | **方案四：内网穿透（ngrok / Cloudflare）** |

---

**方案一：直连 IPv6（零配置）**

- **适用**：双方网络均支持 IPv6（校园网、家庭宽带、手机热点大多已支持）。
- **房主操作**：
  1. 命令行运行 `ipconfig`，记下当前网卡的 **IPv6 地址**（形如 `2402:f000:...`）。
  2. 在项目目录运行 `node server.js`。
  3. 以**管理员**身份执行命令放行防火墙（仅首次）：  
     `netsh advfirewall firewall add rule name="GameServer" dir=in action=allow protocol=TCP localport=3000`
  4. 将地址 `http://[你的IPv6地址]:3000` 发给访客。
- **访客操作**：浏览器粘贴该地址即可访问。

---

**方案二：VPN 内网直连**

- **适用**：双方均连接同一校园 VPN。
- **房主操作**：
  1. 连接 VPN 后，运行 `ipconfig`，查看 VPN 虚拟网卡的 **IPv4 地址**（如 `10.x.x.x`）。
  2. 运行 `node server.js`，放行防火墙（同方案一第3步）。
  3. 将 `http://你的VPN-IP:3000` 发给访客。
- **访客操作**：同样连接 VPN，浏览器输入该地址访问。

---

**方案三：Tailscale 虚拟组网（推荐复杂网络）**

- **适用**：任何网络环境，无需公网 IP。
- **双方操作**：
  1. 下载安装 [Tailscale](https://tailscale.com/)（免费），使用同一账号登录。
  2. 登录后每台设备获得 `100.x.x.x` 虚拟 IP。
  3. 房主运行 `node server.js`，将虚拟 IP（如 `100.64.2.3`）发给访客。
  4. 访客浏览器访问 `http://[100.64.2.3]:3000`。

---

**方案四：内网穿透（ngrok 示例）**

- **适用**：不愿安装任何软件，愿使用第三方服务。
- **房主操作**：
  1. 注册 ngrok 并下载客户端。
  2. 运行 `node server.js`。
  3. 另开命令行执行 `ngrok tcp 3000`，获得类似 `0.tcp.ngrok.io:12345` 的地址。
  4. 将完整地址（包含 `http://`）发给访客。
- **访客操作**：浏览器访问该地址。

> ⚠️ 免费版有流量限制，适合偶尔使用；也可改用 Cloudflare Tunnel（需绑定域名）。

---

**常见问题**
- **访问失败？** 确认 `node server.js` 正常运行，防火墙已放行 3000 端口；IPv6 方案需确保访客网络也支持 IPv6（可用 `test-ipv6.com` 测试）。
- **VPN 无法访问？** 尝试在 VPN 设置中启用“全部流量走 VPN”，或改用 Tailscale。
- **手机热点联机？** 手机热点通常无公网 IP，推荐方案三（Tailscale）或方案四（内网穿透）。

**总结建议**：校内或 VPN 用户优先方案一/二；复杂网络环境首选方案三（Tailscale），五分钟解决网络问题。

### 主题与体验
- **深色/浅色主题**：一键切换，适配不同光照环境，主题偏好自动保存至本地。
- **落子动画**：棋子落下时带有缩放动画，增强反馈感。
- **响应式布局**：针对桌面、平板、手机多种屏幕尺寸优化，确保在小屏设备上也能流畅操作。
- **自动保存**：游戏状态（当前棋盘、历史记录、模式、难度）自动保存在浏览器本地，刷新页面后自动恢复。

## 🛠️ 技术栈

- **前端**：原生 HTML5 + CSS3 + JavaScript（ES6+），无第三方依赖。
- **后端**（在线对战）：Node.js 原生 `http` 模块，无额外框架。
- **数据存储**：前端使用 `localStorage` 保存游戏状态和用户偏好；后端使用内存 `Map` 管理房间数据。
- **通信协议**：RESTful API（JSON），前端通过 `fetch` 与后端交互。
- **开发工具**：VS Code + Live Server（前端调试），Node.js（服务器运行）。

## 🚀 如何开始

**在线体验**：不想本地部署？直接访问 [在线游戏地址](https://thudyq.github.io/Zone-Lock-Chess/) 即可快速开始对局。如需使用在线对战功能，仍需按照下方指引启动本地服务器。

### 本地双人 / 人机对战
1. 克隆仓库或下载所有文件（`index.html`, `style.css`, `script.js`）。
2. 直接用浏览器打开 `index.html` 即可开始游戏（推荐使用 Live Server 或任意静态服务器以避免跨域问题）。

### 在线对战
1. 确保已安装 [Node.js](https://nodejs.org/)（v16 及以上）。
2. 在项目目录下运行：
   ```bash
   node server.js
   ```
   服务器默认监听端口 `3000`，如果端口被占用会自动尝试 3001~3010。
3. 打开浏览器访问 `http://localhost:3000`，即可使用在线对战功能。
4. 若需与远程朋友联机，请参考上方 **联机指南** 选择合适的网络配置方案。

> **提示**：在线对战模式下，前端会自动探测本地服务器地址（localhost 或 127.0.0.1 的 3000~3010 端口），通常无需手动设置。如需指定服务器地址，可在页面顶部“后端地址”输入框中配置。

# Super AI Zone Lock Chess

纯静态版封域棋，在原网页 Easy / Medium / Hard / Expert 的基础上增加了
`Master（MCTS）` 难度。Master 是 `mcts.py` 与 `chess_new.py` 的 JavaScript
BigInt 移植版，在 ES module Web Worker 中运行，不需要 Python、Node 后端或数据库。

## 运行与部署

模块 Worker 需要通过 HTTP(S) 加载。开发时可在本目录运行任意静态服务器，例如：

```powershell
python -m http.server 8080
```

访问 `http://localhost:8080`。生产环境可直接把本目录发布到 GitHub Pages、
Cloudflare Pages、Netlify 或任意静态文件服务器。Master 人机对战不使用原项目的
在线对战后端；在线房间功能仍需要原 Node 服务。
