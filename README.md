

# 封域棋 Zone Lock Chess

一个基于 HTML + CSS + JavaScript 开发的原创棋类网页游戏。

---

# 一、游戏规则

## 基本规则

在一个 N × N 的棋盘中，两名玩家轮流落子。

- 黑棋先手

- 棋子一旦落下不可移动

- 同一格只能落子一次

---

## 合法落子条件（规则A）

某个空格对于玩家 P 合法，当且仅当：

```text
该格上下左右四个方向
不存在对方棋子
```

即：

- 可以贴着自己的棋子下

- 不可以贴着对方棋子下

例如：

```text
⚫ .
⚪ .
```

空格同时邻接黑白棋：

```text
⚫ □ ⚪
```

则：

```text
双方都不能下
```

成为永久废点。

---

## 游戏结束

当轮到某玩家时：

```text
若该玩家无合法落点
↓
立即结束游戏
```

然后判断：

### 情况1

对方仍有合法落点：

```text
对方获胜
```

### 情况2

对方也没有合法落点：

```text
平局
```

# 四、人机对战系统

## 已实现

### 游戏模式

支持：

```text
双人对战
人机对战
```

切换。

---

### AI 难度分级

支持四种难度：

```text
简单：完全随机落子
中等：1 层混合启发式评估
困难：Minimax + Alpha-Beta 搜索（深度 2）
专家：Minimax + Alpha-Beta 搜索（深度 3）
```

#### 简单

```text
获取全部合法位置
↓
随机选择
↓
落子
```

#### 中等

使用 V4 混合评估函数：

```text
score = 1.0 × (AI合法步数 - 玩家合法步数)
      + 0.5 × 中心控制奖励
      + 0.4 × 集团奖励
```

#### 困难 / 专家

使用 Minimax + Alpha-Beta 剪枝：

```text
AI 落子 → 模拟玩家回应 → AI 再落子 → ...
```

- 困难：向前搜索 2 层
- 专家：向前搜索 3 层

终局评分：

```text
AI 无路可走且玩家有路可走 → -10000（AI 负）
玩家无路可走且 AI 有路可走 → +10000（AI 胜）
双方均无路可走           → 0（平局）
```

---

### AI思考延迟

```text
300ms
```

模拟思考过程。

---

### AI锁

避免：

```text
AI行动期间
玩家继续点击
```

---

### 定时器清理

避免：

```text

重新开始
↓
旧AI突然落子
```

问题。

# 六、下一阶段计划

---

## 第一阶段：增强AI

### V1（已完成）

贪心AI：

```text
score =
AI合法步数
-
玩家合法步数
```

选择最高分落点。

---

### V2（已完成）

中心控制奖励

```text
score_center = maxDistance - manhattanDistanceToCenter
```

越接近棋盘中心，评分越高，避免AI开局占角。

---

### V3（已完成）

集团奖励

鼓励形成：

```text
日字型
桥接结构
连片结构
```

统计落点周围8格内的己方棋子数作为奖励，提高空间利用率。

---

### V4（已完成）

混合评估函数

```text
score =
  权重1 × (AI合法步数 - 玩家合法步数)
+ 权重2 × 中心控制奖励
+ 权重3 × 集团奖励
```

形成完整启发式AI。

当前权重：

```text
合法步数权重 = 1.0
中心控制权重 = 0.5
集团奖励权重 = 0.4
```

---

### V5（已完成）

Minimax + Alpha-Beta 剪枝搜索

在 V4 评估函数基础上，引入博弈树搜索：

```text
AI 落子 → 模拟玩家最佳回应 → AI 再落子 → ...
```

实现 Alpha-Beta 剪枝优化搜索效率，支持 2 层（困难）和 3 层（专家）搜索深度。

---

## 第二阶段：棋谱系统（已完成）

### 导出棋谱（已完成）

导出为 `.json` 文件，格式：

```json
{
  "version": "1.0",
  "boardSize": 8,
  "moves": [
    { "row": 3, "col": 3, "player": 1 },
    { "row": 4, "col": 4, "player": 2 }
  ]
}
```

文件名：`棋谱_8x8_2026-06-25.json`

---

### 导入棋谱（已完成）

读取 `.json` 文件，校验格式与棋盘大小，自动进入回放模式。

---

### 棋局回放（已完成）

支持：

```text
⏮ 开始
⏴ 上一步
▶ / ⏸ 播放 / 暂停
⏵ 下一步
⏭ 结束
```

可调整回放速度：慢 / 正常 / 快 / 很快。

回放时：

- 禁用棋盘点击与落子
- 当前步在棋谱记录中高亮显示
- 可随时点击“从当前步继续对弈”恢复对弈

### 实时对弈

服务器持有完整棋局状态，客户端通过轮询 `/room-state` 同步：

```text
落子 → POST /move → 服务器校验并更新状态
                ↓
双方轮询 /room-state ← 获取最新局面
```

支持：

- 黑棋先手
- 服务器端校验每一步合法性
- 自动判定胜负
- 再来一局（双方同意后方可重开）
- 对手离开提示
